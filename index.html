5.3 Набор плагинов для Visual Studio Code

Для Visual Studio Code (VS Code) доступны плагины, которые расширяют функциональность редактора кода. Они могут быть посвящены разным темам: повышению продуктивности, форматированию кода, работе с системой контроля версий (Git) или отладке. Ниже представлены некоторые из них.

Содержание

  1. Продуктивность
  2. Форматирование
  3. Контроль версий
  4. Отладка

1. Продуктивность

  1. HTML Preview — экономим время на переключение в браузер

    Когда вы начинаете верстать страницу, пока не написали стили, но вовсю готовите разметку по макету, нужно часто переключать в браузер и проверять, что вы наразмечали.
    Это бывает долго и неудобно, особенно если у вас пока один монитор или вы вообще верстаете на ноутбуке. Плагин HTML Preview решает эту проблему и экономит массу времени — размеченную страницу видно прямо в окне VS Code, и превью меняется сразу после очередного сохранения HTML-файла.

    Расширение HTML Preview можно активировать двумя способами:

    • Переключить предварительный просмотр - ctrl+shift+v или cmd+shift+v
    • Откройте предварительный просмотр сбоку — ctrl+k v или cmd+k v

  2. JavaScript (ES6) Code Snippets — предоставляет сокращения для общих шаблонов кода JavaScript, что ускоряет разработку и избавляет от повторяющегося набора кода.

    Поддерживаемые языки: JavaScript (.js), TypeScript (.ts), JavaScript React (.jsx), TypeScript React (.tsx), HTML (.html), Vue (.vue).

    3. Функции

    Сниппеты включают шаблоны для функций, классов, модулей и других структур.

    • Импорт и экспорт — сниппеты для импорта всего модуля, импорта части модуля с помощью destructing и других задач.
    • Классные помощники — сниппеты для добавления конструктора в класс, создания метода внутри класса, свойства-геттера и свойства-сеттера.
    • Методы — сниппеты для цикла forEach в синтаксисе ES6, для цикла for ... of, для цикла for ... in и других методов.
    • Методы консоли — сниппеты для console.log, console.clear, console.count и других методов.

    Особенности:

    • Сниппеты оптимизированы, чтобы быть короткими и легко запоминаться.
    • Есть возможность изменять и создавать собственные фрагменты кода, адаптируя их к стилю программирования и требованиям проекта.

  3. Todo Highlight — выделяет комментарии к TODO в коде, упрощает поиск задач и напоминаний в больших кодовых базах.

    Некоторые возможности расширения:

    • Настройка ключевых слов и цветов подсветки. Пользователи могут определять свои ключевые слова и назначать им определённые цвета для лучшей категоризации.
    • Подсветка в нескольких файлах. Поддерживается выделение комментариев в разных файлах рабочего пространства.
    • Список задач с возможностью поиска. Интегрируется с палитрой команд для быстрого поиска и списка всех выделенных комментариев в текущем рабочем пространстве.
    • Гибкая конфигурация. Предлагает конфигурацию через файлы настроек, что позволяет разработчикам адаптировать расширение к стандартам команды или личным рабочим процессам.
    • Оптимизированность и скорость. Расширение оптимизировано для скорости и не влияет на общую производительность редактора, даже в больших базах кода.

    Кому подойдёт: разработчикам, которые оставляют заметки в коде, и командам, которым нужно быстро находить задачи или недочёты в проекте.

2. Форматирование

  1. Prettier — автоматически форматирует код, поддерживает множество языков программирования и может быть настроен под стиль проекта.

    Особенности:

    • Поддерживает множество языков программирования и шаблонов, включая JavaScript, TypeScript, Flow, JSX, JSON, CSS, SCSS, Less, HTML, Vue, Angular и другие.
    • Интегрируется с большинством текстовых редакторов и систем контроля версий.

    Функции

    • Автоматическое исправление кода по заданным правилам форматирования. Например, Prettier добавляет нужные отступы, точки с запятыми, разбивает длинные строки на более короткие блоки.
    • Настройка форматирования — ширина отступов, предпочтение одинарных или двойных кавычек, расстановка запятых и требования к переносам строк.
    • Игнорирование определённых файлов и директорий — через файл .prettierignore.
    • Автоматическое форматирование при коммитах — для проектов, использующих системы контроля версий, например Git, можно настроить хуки, которые применяют Prettier к коду перед каждым коммитом.

    Установка

    • Добавить пакет Prettier в проект через npm или yarn.
    • Создать пустой файл конфигурации — это позволяет редакторам и другим инструментам знать, что используется Prettier.
    • Создать файл .prettierignore — он даёт Prettier CLI и редакторам знать, какие файлы не форматировать.

    Настройка

    • Определить стандартное поведение форматтера для проекта — это делается через файл .prettierrc.
    • Настроить правила форматирования — можно изменить распространённые параметры или создать отдельный файл конфигурации.
    • Включить форматирование при сохранении — например, в настройках редактора можно отметить чекбокс, чтобы форматирование происходило автоматически при сохранении файла.

    Интеграция

    • Интегрировать Prettier в текстовый редактор — через плагины и расширения. Например, для Visual Studio Code доступно расширение «Prettier — Code formatter».
    • Настроить хуки для проектов, использующих системы контроля версий, — они будут применять Prettier к коду перед каждым коммитом.
    • Реализовать сценарии форматирования как часть рабочего процесса — это поможет гарантировать, что каждое развёртывание будет автоматически отформатировано.

  2. Path Intellisense — предлагает автодополнение для путей к файлам в коде, что упрощает навигацию и ссылки на файлы внутри проекта.

    3. Функции

    Расширение динамически предлагает пути к файлам и имена каталогов по мере ввода. Это помогает:

    • Ускорить написание адресов файлов в коде, например, при установке специального тега, подразумевающего вставку файла.
    • Минимизировать ошибки из-за неполных или неправильных путей к файлам.

    Важно: хотя VS Code поддерживает автозавершение путей и без расширений, эта возможность ограничена только HTML-, CSS- и JavaScript-файлами. Благодаря Path Intellisense автозавершением путей можно пользоваться при работе с любыми файлами и любыми языками.

    Настройка

    Некоторые настройки расширения:

    • Включить расширение файла в операторах импорта — по умолчанию Path Intellisense удаляет расширение файла, если оператор — оператор импорта. Чтобы включить расширение, нужно установить настройку path-intellisense.extensionOnImport: true.
    • Показывать скрытые файлы — по умолчанию скрытые файлы не отображаются, нужно установить настройку path-intellisense.showHiddenFiles: true.
    • Автоматически добавлять слэш при переходе в папку — по умолчанию автозаполнение не добавляет слэш после каталога, нужно установить настройку path-intellisense.autoSlashAfterDirectory: false.
    • Разрешать абсолютные пути — по умолчанию абсолютные пути разрешаются в корневом пути текущей рабочей области, нужно установить настройку path-intellisense.absolutePathToWorkspace: true.

3. Контроль версий

  1. GitLens— расширяет стандартные возможности Git в VS Code, предоставляет информацию о том, кто и когда вносил изменения, отображает историю коммитов.
    Некоторые функции:
    • аннотации для кода, показывающие автора, дату и сообщение о последнем изменении в каждой строке;
    • навигация по коду, возможность быстро перейти к истории файла или перемещаться по изменениям в файле со временем;
    • сравнение различий между двумя версиями файла.

4. Отладка

  1. Debugger for Chrome — расширение для Visual Studio Code, которое позволяет отлаживать JavaScript-код, запущенный в Google Chrome, прямо из редактора. Расширение работает с использованием протокола Chrome DevTools, который позволяет сопоставлять файлы, загруженные в браузере, с файлами, открытыми в VS Code.

    Функции

    Расширение позволяет:

    • Устанавливать точки останова в исходном коде, включая файлы с включёнными картами источников.
    • Пошагово отлаживать код с помощью кнопок на странице Chrome.
    • Отлаживать скрипты eval, теги скриптов и скрипты, которые добавляются динамически
    • Видеть полный стек вызовов при отладке.

    Настройка

    Для работы отладчика нужно создать файл конфигурации launch.json в корневом каталоге проекта. Можно создать этот файл вручную или VS Code создаст его автоматически, если при запуске проекта файл не существует.

    Некоторые особенности настройки:

    • Режим launch — отладчик запускает экземпляр Chrome, запущенный по указанному файлу или URL. Если указан URL, нужно указать каталог, из которого обслуживаются файлы (можно использовать абсолютный путь или путь с использованием ${workspaceFolder} — папки, открытой в VS Code).
    • Режим attach — отладчик присоединяется к уже работающему экземпляру Chrome. Для этого нужно запустить Chrome с включённой удалённой отладкой.

    Запуск отладчика можно выбрать в выпадающем списке на панели отладки в VS Code и нажать кнопку Play или F5.


  2. Turbo Console Log — расширение для Visual Studio Code, которое автоматизирует создание сообщений console.log в коде. Оно упрощает процесс отладки, избавляя от ручного добавления журнальных сообщений.

    Функции

    • Автоматическое создание сообщения console.log для выделенной переменной. Расширение генерирует сообщение, которое включает имя переменной и её значение.
    • Поддержка нескольких переменных — можно отлаживать несколько переменных одновременно.
    • Возможность включать в сообщение имя функции или класса, который содержит переменную. Это добавляет контекс
    • Горячие клавиши для управления сообщениями: комментарий, uncomment, удаление. Например:
      • Ctrl + Alt + L — создаёт console.log для выделенной переменной.
      • Ctrl + Alt + D — удаляет все console.log из файла

    Настройка

    После установки расширения можно настроить его параметры под свои нужды.
    Некоторые возможности:

    • Изменение префикса для сообщения. Расширение поддерживает разные символы и символы в качестве префиксов.
    • Включение номеров файла и строки.
    • Изменение разделителя.
    • Автоматическое добавление точек с запятой в конце сообщений, что избавляет от необходимости делать это вручную.

    Некоторые настройки расширения:

    • Включить расширение файла в операторах импорта — по умолчанию Path Intellisense удаляет расширение файла, если оператор — оператор импорта. Чтобы включить расширение, нужно установить настройку path-intellisense.extensionOnImport: true.
    • Показывать скрытые файлы — по умолчанию скрытые файлы не отображаются, нужно установить настройку path-intellisense.showHiddenFiles: true.
    • Автоматически добавлять слэш при переходе в папку — по умолчанию автозаполнение не добавляет слэш после каталога, нужно установить настройку path-intellisense.autoSlashAfterDirectory: false.
    • Разрешать абсолютные пути — по умолчанию абсолютные пути разрешаются в корневом пути текущей рабочей области, нужно установить настройку path-intellisense.absolutePathToWorkspace: true.

4. Отладка

  1. Debugger for Chrome — расширение для Visual Studio Code, которое позволяет отлаживать JavaScript-код, запущенный в Google Chrome, прямо из редактора. Расширение работает с использованием протокола Chrome DevTools, который позволяет сопоставлять файлы, загруженные в браузере, с файлами, открытыми в VS Code.

    Функции

    Расширение позволяет:

    • Устанавливать точки останова в исходном коде, включая файлы с включёнными картами источников.
    • Пошагово отлаживать код с помощью кнопок на странице Chrome.
    • Отлаживать скрипты eval, теги скриптов и скрипты, которые добавляются динамически
    • Видеть полный стек вызовов при отладке.

    Настройка

    Для работы отладчика нужно создать файл конфигурации launch.json в корневом каталоге проекта. Можно создать этот файл вручную или VS Code создаст его автоматически, если при запуске проекта файл не существует.

    Некоторые особенности настройки:

    • Режим launch — отладчик запускает экземпляр Chrome, запущенный по указанному файлу или URL. Если указан URL, нужно указать каталог, из которого обслуживаются файлы (можно использовать абсолютный путь или путь с использованием ${workspaceFolder} — папки, открытой в VS Code).
    • Режим attach — отладчик присоединяется к уже работающему экземпляру Chrome. Для этого нужно запустить Chrome с включённой удалённой отладкой.

    Запуск отладчика можно выбрать в выпадающем списке на панели отладки в VS Code и нажать кнопку Play или F5.


  2. Turbo Console Log — расширение для Visual Studio Code, которое автоматизирует создание сообщений console.log в коде. Оно упрощает процесс отладки, избавляя от ручного добавления журнальных сообщений.

    Функции

    • Автоматическое создание сообщения console.log для выделенной переменной. Расширение генерирует сообщение, которое включает имя переменной и её значение.
    • Поддержка нескольких переменных — можно отлаживать несколько переменных одновременно.
    • Возможность включать в сообщение имя функции или класса, который содержит переменную. Это добавляет контекс
    • Горячие клавиши для управления сообщениями: комментарий, uncomment, удаление. Например:
      • Ctrl + Alt + L — создаёт console.log для выделенной переменной.
      • Ctrl + Alt + D — удаляет все console.log из файла

    Настройка

    После установки расширения можно настроить его параметры под свои нужды.
    Некоторые возможности:

    • Изменение префикса для сообщения. Расширение поддерживает разные символы и символы в качестве префиксов.
    • Включение номеров файла и строки.
    • Изменение разделителя.
    • Автоматическое добавление точек с запятой в конце сообщений, что избавляет от необходимости делать это вручную.
